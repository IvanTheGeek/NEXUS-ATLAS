Design System

Canonical tokens and components for the ATLAS model viewer. All components reference these values — nothing is invented at the component level. Target: tablet minimum, laptop/desktop typical. Base text: 16 px (browser default), user-adjustable via size control in nav.

The neutral shell palette. Backgrounds, surfaces, borders, and text scale for the viewer itself. Switches between light and dark modes.

Typography

Two typefaces. IBM Plex Sans for all UI text. IBM Plex Mono for field names, property values, code, and anything requiring character-level alignment. All sizes are rem — relative to the user's chosen text size.

Badges

Non-interactive classification labels. Every badge has a specific semantic role — they are not decorative. Interactive triggers (clickable pills that open popovers) use <button> but share the same visual treatment.

Context Group — coarse cluster label
App Runtime Business UX / Interaction

Dark solid pill. Identifies the coarse group a column belongs to. Always leftmost in the classification row.

Bounded Context — ownership boundary
ApplicationLifecycle RuntimeOrchestration Domain UX

Light outlined pill. Identifies which bounded context owns the meaning of this step.

Lens — cross-cutting view filter
Lifecycle Runtime Screen AEM

Faintest badge. Identifies which lens shows this column. Multiple lenses possible per column.

Column Kind — square tag, semantic color
SLICE COMMAND EVENT VIEW

Square corner tag. Always right-aligned in the classification row. Color matches the entity type palette.

Actor
Driver Modeler System Automation

Blue tint = human actor. Neutral = system/automation actor.

GWT Status — square tag
Specified Draft No spec

Shown in the GWT zone when GWT is collapsed. Green = spec exists and is complete. Yellow = in progress. Grey = infrastructure slice, no spec expected.
